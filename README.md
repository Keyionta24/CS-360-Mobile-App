# CS 360 Mobile App Portfolio Artifact

## Inventory App Summary

For this course, I created an inventory mobile app. The goal of the app was to help users keep track of items in a simple and organized way. The app lets users log in, view inventory items, add new items, edit items, delete items, and use SMS permission for alerts when inventory reaches zero.

This app was designed to help users who need a quick way to manage inventory. The main user need was organization. Users should be able to see what items they have, update item details, and know when an item is low or out of stock.

## Screens and Features

The app needed several screens to support the user. These screens included a login screen, register screen, inventory list screen, add item screen, edit item screen, and notification permission screen. Each screen had a clear purpose so the app would be easy to understand.

My UI design kept users in mind by making the screens simple and direct. I used clear buttons, easy labels, and a layout that was not overcrowded. The inventory screen helped users see their items, while the add and edit screens made it easy to change information. I think the design was successful because users could move through the app without needing a lot of extra instructions.

## Coding Process

I approached the coding process one step at a time. I started by creating the main screens first. Then I connected the buttons, activities, and database features. Working in smaller steps helped me understand what each part of the app was doing.

Some strategies I used were testing each screen after making changes, checking my XML files, and making sure my Java files matched the correct activities. These strategies can help me in the future because breaking a project into smaller parts makes coding easier to manage.

## Testing

I tested the app by running it in the Android emulator and checking each screen. I made sure the login and register screens opened correctly. I also tested the inventory list, add item screen, edit item screen, delete option, and notification permission screen.

Testing is important because an app can look finished but still have problems. Testing helped me find small issues and fix them before submitting the project. It also showed me that testing should be done during the whole project, not only at the end.

## Challenges

One challenge I had was making sure all of the screens worked together. At first, it was confusing to connect the layout files, Java files, database helper, and Android Manifest. I had to slow down and check each part carefully.

I also had to work through Android permission issues for the SMS notification feature. This helped me understand that mobile app development is not just about writing code. It also includes handling permissions, testing on a device, and making sure the user experience works correctly.

## Area of Success

I was most successful with the inventory flow of the app. The app allows users to move between screens and manage inventory items in a clear way. I was also proud of getting the notification permission screen to work because it showed that I could connect Android permissions to app features.

Overall, this project helped me understand the full mobile app development process. I learned how planning, UI design, coding, testing, and final submission all connect together.
